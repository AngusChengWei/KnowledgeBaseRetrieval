# 需求变更方案（CR）
## 企业 AI 知识助手 —— 新增「URL 知识导入模块」

---

## 1. 变更基本信息

- 变更名称：URL 知识导入模块
- 变更类型：功能新增（Feature Add）
- 影响版本：v1.1
- 影响范围：知识库管理 / 数据导入 / RAG
- 是否破坏现有功能：否
- 是否强制启用：否（管理员配置）

---

## 2. 变更背景

当前系统仅支持通过 **文件上传（PDF / Word / 文本）** 构建知识库，但在企业实际场景中：

- 大量知识以 **在线页面（URL）** 形式存在
- 包括 Wiki、内部门户、接口文档、制度公告等
- 需要人工复制、整理，成本高、易出错

因此需要新增 **“通过 URL 一键导入知识库”** 能力，以降低企业知识接入门槛。

---

## 3. 变更目标

### 3.1 功能目标

- 支持管理员通过 URL 导入网页内容
- 自动抽取网页正文文本
- 将网页内容构建为可问答的知识库数据
- 支持权限控制与来源溯源

### 3.2 非目标（本次不做）

- 不支持全站爬取
- 不支持自动跟链
- 不支持登录态模拟
- 不支持 JS 动态页面深度渲染

---

## 4. 功能范围说明

### 4.1 支持的 URL 类型

- http / https 协议
- 单页面 URL
- 静态或弱动态页面

### 4.2 不支持的 URL 类型

- 需要登录才能访问的页面（首期）
- 文件下载链接（如 .zip）
- 视频 / 音频页面

---

## 6. 功能流程设计

### 6.1 管理端操作流程

1. 进入「知识库管理」
2. 选择目标知识库
3. 点击「URL 导入」
4. 输入 URL
5. 配置导入参数
6. 提交导入任务

---

### 6.2 系统执行流程


URL 输入
↓
合法性校验（协议 / 域名）
↓
页面抓取（HTTP GET）
↓
HTML 解析
↓
正文内容抽取
↓
文本清洗
↓
内容切分（Chunk）
↓
Embedding
↓
写入知识库


---

## 7. URL 合法性与安全控制

### 7.1 URL 校验规则

- 仅允许 http / https
- 禁止内网地址（127.0.0.1 / 10.x / 192.168.x）
- 可配置域名白名单

### 7.2 域名白名单（可选）

```yaml
allowed_domains:
  - wiki.company.com
  - docs.company.com
8. 内容抽取规则
8.1 必须保留内容
页面主标题
正文段落
列表、表格中的文本
8.2 必须移除内容
Header / Footer
导航菜单
广告
评论区
推荐阅读

  "content": "正文文本",
  "metadata": {
    "department": "string",

  }

10. 知识库行为规则
URL 导入内容与文件导入内容一视同仁
参与统一 RAG 检索
权限继承所属知识库
问答结果需标注来源 URL
11. 问答展示要求
11.1 回答结果展示
正文答案（文本）
来源链接（URL）
所属知识库名称

示例：

来源：
https://wiki.company.com/xxx
12. 数据库变更说明
12.1 新增字段（如统一表）
ALTER TABLE knowledge_unit ADD COLUMN source_type TEXT;
ALTER TABLE knowledge_unit ADD COLUMN source_ref TEXT;
13. 异常处理
场景	行为
URL 无法访问	导入失败
内容为空	导入失败
非法域名	拒绝导入
超时	终止任务
14. 日志与审计
记录 URL 导入日志
记录导入人、时间、目标知识库
可供超级管理员审计
15. 验收标准
合法 URL 可成功导入
内容可被 AI 问答命中
权限控制准确
来源 URL 可追溯
不影响现有功能
16. 风险说明
页面结构变化可能导致抽取失败
内容时效性需人工重新导入
首期不保证复杂页面结构识别
